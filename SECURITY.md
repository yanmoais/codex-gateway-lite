# Security Policy

## Supported versions

当前只维护 `main` 分支和最新发布版本。

## Reporting a vulnerability

请不要在公开 issue 里贴安全漏洞、API Key、auth 文件、私有 Base URL、完整日志或用户配置。

推荐做法：

1. 使用 GitHub private vulnerability reporting（如果仓库已开启）。
2. 或者创建不含敏感细节的 issue，请维护者提供私下沟通方式。

报告中可以包含：

- 受影响版本或 commit。
- 操作系统。
- 最小复现步骤。
- 影响范围。
- 已经做过的规避措施。

维护者会尽快确认影响范围，并在修复后公开披露必要信息。
